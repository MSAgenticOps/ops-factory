# IT外包服务合同审核基线 (Baseline)

> 本基线适用于IT运维外包、托管服务、应用开发外包等合同审核。
> 审核时应结合具体业务场景和战略考量进行判断。

---

## 🔴 红线条款（绝不允许）

以下条款触及公司核心利益底线，**必须拒绝或要求删除/修改**，无论对方如何坚持。

### 1. 责任与赔偿

- **无限责任**：供应商不得要求我方承担无上限的赔偿责任；同样，我方也不应承诺无限责任
- **间接损失**：不得承诺对间接损失、利润损失、商誉损失、惩罚性赔偿承担责任
- **责任上限缺失**：合同必须明确责任赔偿上限（通常不超过合同总价的100%或过去12个月已付费用）
- **单方免责**：供应商不得通过格式条款单方面免除其重大过失或故意行为的责任

### 2. 知识产权与数据

- **IP归属不明**：我方业务数据、客户数据的所有权必须明确归我方所有
- **衍生作品归属**：基于我方数据或业务定制开发的成果，知识产权应归我方所有或至少获得永久、不可撤销的使用许可
- **数据留置**：合同终止后，供应商不得以任何理由留置我方数据；必须在约定期限内完成数据迁移和销毁
- **数据出境**：未经书面同意，不得将我方数据传输至境外或存储于境外服务器

### 3. 保密与合规

- **保密期限过短**：保密义务期限不得短于合同终止后3年（敏感数据应为5年或永久）
- **审计权缺失**：我方必须保留对供应商的审计权，包括安全审计、合规审计
- **转包限制**：核心服务不得转包；非核心服务转包须经我方书面同意，且转包方须承担同等保密义务
- **合规责任转嫁**：供应商不得将监管合规责任（如数据保护、行业准入）完全转嫁给我方

### 4. 服务范围

- **范围无边界**：服务范围必须有明确边界，不得使用"包括但不限于"等开放性表述定义核心义务
- **软件开发纳入运维**：运维外包合同不应包含软件功能开发、重大架构改造等开发性工作
- **硬件采购责任**：运维服务不应包含硬件采购、更换的成本承担责任
- **第三方费用**：不得承担第三方许可证费用、云服务费用等非人工服务成本

### 5. 终止与退出

- **锁定条款**：不得接受超过3年的强制锁定期，且必须保留提前终止权（至少90天通知）
- **高额违约金**：提前终止的违约金不得超过剩余合同期限费用的30%
- **过渡期缺失**：必须约定合理的过渡期（不少于90天）和知识转移义务
- **数据扣押**：供应商不得以费用争议为由拒绝返还数据或配合过渡

## 🟠 橘红条款（战略可商量）

以下条款可能导致我方利益受损，但**基于战略考量可以协商**。若接受，需高层审批并记录商业理由。

### 1. SLA与绩效

- **响应时间放宽**：
  - 标准要求：P1 ≤ 15分钟，P2 ≤ 30分钟，P3 ≤ 4小时，P4 ≤ 1工作日
  - 可商量范围：P1可放宽至30分钟，P2可放宽至1小时（需降低服务费）
- **恢复时间放宽**：
  - 标准要求：P1 ≤ 2小时，P2 ≤ 4小时
  - 可商量范围：P1可放宽至4小时（需配套服务信用机制）
- **SLA豁免期**：过渡期内（最长6个月）可适当放宽SLA要求，但需明确达标时间表
- **服务信用上限**：服务信用总额可协商（标准为月度服务费的15-25%）

### 2. 费用与付款

- **付款周期**：
  - 标准要求：月结，发票后30天付款
  - 可商量范围：季度预付、45天账期（需相应折扣）
- **价格调整机制**：可接受年度价格调整，但涨幅不超过CPI或3%（取较低者）
- **超范围工作**：可接受一定额度的弹性工作量（不超过合同总量的10%），但超出部分需按约定费率另计
- **差旅费用**：现场服务差旅费可由我方承担，但需事先审批且设置年度上限

### 3. 人员与资源

- **关键人员更换**：可接受供应商更换关键人员，但需提前30天通知并确保同等资质
- **人员驻场比例**：可根据实际需求调整驻场/远程比例，但关键岗位应保持稳定
- **培训义务**：可接受我方承担部分培训成本（如特殊系统、专有技术）
- **知识产权许可**：供应商通用工具/框架的知识产权可归供应商，但我方应获得合同期内免费使用许可

### 4. 责任分配

- **责任上限调整**：责任上限可在合同总价50%-150%之间协商
- **保险要求**：可放宽供应商保险金额要求（但不得低于单次事故100万元）
- **不可抗力范围**：可适当扩大不可抗力定义，但必须排除供应商系统故障、人员罢工等

## 🔵 蓝色条款（措辞风险）

以下条款本身不构成重大风险，但**措辞模糊可能引发争议**，建议修改以消除歧义。

### 1. 定义与范围

- **"合理"/"适当"**：避免使用"合理努力"、"适当措施"等主观表述，应量化或明确标准
  - ❌ "供应商应采取合理措施保护数据安全"
  - ✅ "供应商应采取符合ISO 27001标准的安全措施"
- **"包括但不限于"**：在义务条款中避免使用，改为穷尽式列举
  - ❌ "服务范围包括但不限于系统监控、故障处理..."
  - ✅ "服务范围限于以下内容：1) 系统监控；2) 故障处理..."
- **"及时"/"尽快"**：应明确具体时限
  - ❌ "供应商应及时响应故障"
  - ✅ "供应商应在收到通知后15分钟内响应故障"

### 2. 权利义务

- **"协助"/"配合"**：明确协助的具体内容和边界
  - ❌ "供应商应协助我方进行系统升级"
  - ✅ "供应商应提供技术文档、参与技术评审会议、执行我方批准的升级方案"
- **"最大努力"**：避免使用，改为结果导向的义务
  - ❌ "供应商应尽最大努力确保系统可用性"
  - ✅ "供应商应确保系统月度可用性不低于99.9%"
- **"重大变更"**：需定义何为"重大"
  - 建议定义：影响超过10%用户、停机超过1小时、涉及核心业务流程变更

### 3. 时间与期限

- **"工作日"/"日历日"**：统一使用并明确定义
  - 建议："工作日"指中国大陆法定工作日，不含周末及国家法定节假日
- **"合同期限届满后"**：明确是否自动续约及续约条件
  - 建议：明确约定"除任一方提前60天书面通知不续约外，合同自动续展1年"
- **"不可抗力持续期间"**：需设置最长期限和终止权
  - 建议："不可抗力持续超过30天，任一方有权终止合同且互不承担违约责任"

### 4. 费用与支付

- **"按实际发生"**：需明确计量标准和核验方式
  - ❌ "差旅费按实际发生金额报销"
  - ✅ "差旅费按公司差旅标准报销，机票经济舱、酒店不超过500元/晚"
- **"市场价格"**：需明确参考来源
  - 建议：引用具体的价格指数或约定价格确定机制
- **"额外费用"**：需穷尽列举可能产生额外费用的情形

### 5. 争议解决

- **管辖法院**：明确指定具体法院
  - ❌ "由有管辖权的法院管辖"
  - ✅ "由甲方住所地人民法院管辖"或"提交北京仲裁委员会仲裁"
- **通知方式**：明确有效送达方式和地址
  - 建议：约定邮寄地址、电子邮件，并规定送达生效时间
