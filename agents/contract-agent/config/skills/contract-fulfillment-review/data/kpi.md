# IT外包运维服务合同 — 关键绩效指标 (KPI)

> **合同编号**: ITOM-2025-0078
> **合同期限**: 2025-07-01 至 2028-06-30
> **服务范围**: 甲方核心业务系统及办公系统的基础设施运维服务
> **考核周期**: 按月考核，按季度结算服务信用

---

## 1. 系统可用性

### 1.1 核心业务系统可用性
- **目标值**: 月度可用性 ≥ 99.9%
- **计算方式**: (月度总分钟数 - 计划外停机分钟数) / 月度总分钟数 × 100%
- **考核范围**: ERP系统、CRM系统、核心数据库集群、支付网关
- **违约条款**: 每低于目标0.1%，扣减当月服务费的2%；低于99.5%视为重大违约

### 1.2 非核心系统可用性
- **目标值**: 月度可用性 ≥ 99.5%
- **考核范围**: OA系统、邮件系统、内部wiki、文件共享服务
- **违约条款**: 每低于目标0.5%，扣减当月服务费的1%

### 1.3 计划内维护窗口
- **要求**: 每月计划维护窗口不超过8小时，须提前5个工作日通知甲方
- **时间限制**: 仅允许在周六 22:00 - 次日周日 06:00 进行

---

## 2. 故障响应与处理

### 2.1 响应时间
| 故障等级 | 定义 | 响应时间要求 |
|---------|------|------------|
| P1 - 紧急 | 核心业务系统完全不可用 | ≤ 15 分钟 |
| P2 - 严重 | 核心功能受限或性能严重下降 | ≤ 30 分钟 |
| P3 - 一般 | 非核心功能异常，有临时解决方案 | ≤ 4 小时 |
| P4 - 轻微 | 不影响业务的小问题 | ≤ 1 个工作日 |

### 2.2 解决时间
| 故障等级 | 解决时间要求 |
|---------|------------|
| P1 - 紧急 | ≤ 2 小时 |
| P2 - 严重 | ≤ 4 小时 |
| P3 - 一般 | ≤ 8 小时 |
| P4 - 轻微 | ≤ 3 个工作日 |

### 2.3 故障复发率
- **要求**: 同一根因故障30天内复发率 ≤ 5%
- **违约条款**: 复发率超过10%，需提交专项整改报告并扣减当月服务费的3%

---

## 3. 服务台指标

### 3.1 一次解决率
- **目标值**: ≥ 70%
- **定义**: 用户首次联系服务台即解决的工单占比

### 3.2 用户满意度
- **目标值**: ≥ 90%（满意 + 非常满意）
- **评估方式**: 每月随机抽样不少于100份工单进行回访
- **违约条款**: 连续两个月低于85%，甲方有权要求更换服务台负责人

### 3.3 电话接通率
- **目标值**: 20秒内接通率 ≥ 95%
- **工作时间**: 工作日 08:00 - 20:00，非工作时间提供值班电话

### 3.4 工单处理及时率
- **目标值**: ≥ 95%的工单在SLA规定时间内关闭

---

## 4. 变更管理

### 4.1 变更成功率
- **目标值**: ≥ 95%
- **定义**: 变更实施后未导致新增故障或服务中断的变更占比

### 4.2 紧急变更比例
- **目标值**: ≤ 10%
- **定义**: 紧急变更数量占总变更数量的比例
- **违约条款**: 连续两个月超过15%，需提交变更流程优化方案

### 4.3 变更回退率
- **目标值**: ≤ 5%
- **定义**: 变更实施后需要回退的变更占比

---

## 5. 安全合规

### 5.1 安全事件
- **要求**: 高危安全事件 = 0，中危安全事件 ≤ 2次/月
- **定义**: 高危指数据泄露、系统被入侵、勒索软件感染等；中危指未授权访问尝试、漏洞未及时修复等
- **违约条款**: 发生1次高危安全事件，扣减当季服务费的10%

### 5.2 补丁合规率
- **目标值**: 关键安全补丁30天内部署率 ≥ 95%
- **考核方式**: 每月末统计未部署的已发布关键补丁数量

### 5.3 备份成功率
- **目标值**: ≥ 99%
- **考核范围**: 所有生产数据库及核心应用的每日备份任务
- **恢复演练**: 每季度至少完成1次备份恢复演练，RTO ≤ 4小时

---

## 6. 人员管理

### 6.1 关键人员稳定性
- **目标值**: 关键岗位人员季度留存率 ≥ 90%
- **关键岗位**: 项目经理、DBA、网络工程师、安全工程师
- **违约条款**: 关键人员离职须提前30天通知，并在15天内提供同等资质替换人选

### 6.2 驻场出勤率
- **目标值**: ≥ 95%
- **考核方式**: 按月统计驻场人员实际出勤天数 / 应出勤天数

### 6.3 培训与认证
- **要求**: 驻场人员每年不少于40小时专业培训
- **认证要求**: DBA须持有Oracle OCP或同等认证，安全工程师须持有CISSP或CISP

---

## 7. 报告与沟通

### 7.1 月度运维报告
- **交付时间**: 每月第5个工作日之前
- **内容要求**: 包含可用性统计、故障分析、SLA达成率、变更汇总、安全态势

### 7.2 季度服务回顾会议
- **频率**: 每季度至少1次
- **参与人员**: 乙方项目经理及技术骨干、甲方IT负责人
- **内容**: KPI达成分析、重大事件回顾、改进计划

### 7.3 重大事件报告
- **时效**: P1故障发生后24小时内提交根因分析报告 (RCA)
- **内容**: 时间线、根因、影响范围、临时措施、长期改进方案
