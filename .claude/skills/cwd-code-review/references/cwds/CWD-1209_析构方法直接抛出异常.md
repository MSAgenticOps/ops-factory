# CWD-1209 析构方法直接抛出异常

**描述**
从析构函数中抛出异常会导致std::terminate。

**语言: **CPP

**严重等级**
提示

**cleancode特征**
可维护

**示例**
**案例1: 析构方法中抛出异常**
**语言: **CPP

**描述**
析构函数默认自带noexcept属性，如果析构函数抛出异常，会直接导致`std::terminate`。

**反例**
```cpp
class A {
public:
    ...
    ~A()
    {
        throw std::exception(); // 不符合
    }
};
int main()
{
    A a;
    return ;
}?
```

**正例**
```cpp
class A {
public:
    ...
    ~A()
    {
        try {
            throw std::exception(); // 符合
        } catch (const std::exception &ex) {
            ...
        }
    }
    void Foo()
    {
        throw std::exception();
    }
};
```

**修复建议**
如果一定要在析构函数中抛异常，应在析构函数内处理完此异常，不能抛出函数之外。
