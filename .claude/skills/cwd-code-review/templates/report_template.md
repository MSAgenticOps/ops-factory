## 附录：报告格式说明

### 发现格式模板

每个发现应按以下格式组织：

```markdown
### [CWD-XXXX]: 缺陷类别名称
**严重程度:** [严重|高|中|低|提示]
**影响文件数:** N
**Clean Code特性:** [特性1, 特性2]

#### 发现描述
- **文件:** path/to/file: start-end
- **代码:**
  ```{language}
  代码片段（3-10行）
  ```
- **描述:** 缺陷的详细说明
- **修复建议:** 包含示例的修复方案
```

### 发现格式示例

```markdown
### CWD-1002: 内存分配大小未限制
**严重程度:** 高
**影响文件数:** 3
**Clean Code特性:** 防御性编程, 输入验证

#### 发现: src/utils.c:45-52
- **代码:**
  ```c
  void *ptr = malloc(n);  // n 来自外部输入
  if (ptr == NULL) {
      return ERROR;
  }
  ```
- **描述:** 内存分配大小由外部输入控制，未经验证。
- **修复建议:**
  ```c
  #define MAX_ALLOC_SIZE 1048576  // 1MB 限制

  if (n == 0 || n > MAX_ALLOC_SIZE) {
      return ERROR;
  }
  void *ptr = malloc(n);
  ```

### 严重程度分类
- **严重**: 安全漏洞或可能导致崩溃
- **高**: 重大缺陷，影响范围广
- **中**: 重要的代码质量问题
- **低**: 小的改进建议
- **提示**: 信息性备注
